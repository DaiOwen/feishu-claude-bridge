## WebSocket 长连接细节

### 连接建立

1. `Client.start()` → 调用飞书 REST API `POST /open-apis/ws/v2/gen`
2. 飞书返回 WebSocket URL (含一次性 token)
3. 建立 WSS 连接

### 协议帧格式

飞书使用 Protobuf 编码的二进制帧：

```
Frame {
    headers: [{key: "type", value: "event"}, ...]
    method: DATA | CONTROL
    service: <service_id>
    SeqID: <seq>
    LogID: <log_id>
    payload: <bytes>
}
```

### 消息类型

| Type | 含义 |
|------|------|
| `event` | 业务事件（消息接收等） |
| `ping` | 心跳 |
| `pong` | 心跳响应 |
| `card` | 卡片回调 |

### 重连机制

SDK 内置指数退避重连：
- 首次重连：随机 0-30s 抖动
- 后续：固定间隔 (默认 120s)
- `reconnect_count = -1` 表示无限重连

### 合包处理

单条大消息可能被拆成多个 Frame，SDK 自动根据 `sum` 和 `seq` 重组。

## 飞书 SDK 事件分发

```python
# 事件处理器注册 (编译时确定类型)
EventDispatcherHandler.builder(encrypt_key, verification_token)
    .register_p2_im_message_receive_v1(callback)   # 注册消息接收处理器
    .register_p2_customized_event("my_event", cb)   # 注册自定义事件
    .build()

# 运行时流程
WebSocket 收到 binary frame
  → 解 Protobuf Frame
    → 检查 FrameType: DATA
      → 检查 MessageType: EVENT
        → 调用 event_handler._do_without_validation(payload)
          → JSON 反序列化为 P2ImMessageReceiveV1
            → 查找 processor: "p2.im.message.receive_v1"
              → 调用 callback(event)
```

## Claude Code CLI 管道模式

```bash
claude -p "你的提示词"
```

- `-p` 表示管道模式（pipe mode）
- 读取参数作为 prompt，输出到 stdout
- 不启动交互式 REPL，执行一次后退出
- 继承当前用户的 Claude Code 配置（模型、权限、MCP 等）
- 有完整的上下文记忆（跨消息）

与普通 ChatGPT API 的关键区别：
- **不仅是文本生成** — 可以读文件、写代码、执行命令
- **有上下文** — 记住项目结构、之前的对话
- **持久化产出** — 写的文件留在你的电脑上

## 安全模型

```
┌────────────┐      ┌──────────────┐      ┌────────────┐
│  手机飞书   │─────▶│  飞书服务器    │─────▶│  你的电脑   │
│            │ TLS  │  (TLS 传输)   │ TLS  │  (本地)    │
└────────────┘      └──────────────┘      └────────────┘

端到端安全:
  飞书 ↔ 你的电脑: TLS 1.3 加密
  凭证: 不在代码中(用环境变量), .gitignore 已排除

攻击面:
  - 飞书账号被盗 → 攻击者可以给机器人发消息
  - 本地电脑被入侵 → 攻击者可以操作 Claude Code
  - 飞书服务器被攻破 → 影响所有飞书用户

建议:
  - 在 bridge.py 中添加 sender_open_id 白名单
  - 限制 Claude Code 操作权限
```

## 扩展可能性

当前架构支持自然扩展：

```
bridge.py (消息入口)
├── 消息路由 ──▶ claude -p (通用问答/代码)
├── 消息路由 ──▶ 自定义处理器 1 (定时任务)
├── 消息路由 ──▶ 自定义处理器 2 (文件管理)
└── 消息路由 ──▶ 自定义处理器 3 (系统监控)

未来可以加:
  - 卡片消息交互
  - 定时推送 (你的服务器状态日报)
  - 多用户权限控制
  - 对话上下文持久化
```
